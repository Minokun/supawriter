## 一、产品背景

P1 完成后，SupaWriter 已具备完整的付费价值体系（SEO优化、批量生成、写作Agent、数据看板）和商业化基础设施。P2 的目标是**从工具升级为平台**，构建生态闭环，提升用户粘性和ARPU。

**P2 核心命题**: 让 SupaWriter 成为用户工作流中不可替代的一环

**来自 Research Report 的关键发现**:
- "一键发布"完成"写→发"闭环，是用户最后一公里需求（§6.1 P2）
- 私有知识库是企业级需求，高ARPU（§6.1 P2）
- 团队协作切入B端市场，ARPU ¥199+/人/月（§6.2 策略三）
- 定位演进：自媒体AI写作助手 → 内容创作操作系统（§11.4）

### F13: 一键发布

#### 3.1.1 功能概述

对接主流内容平台API，用户在SupaWriter中生成文章后可一键发布到公众号、头条号、知乎等平台，无需手动复制粘贴。

#### 3.1.2 用户故事

> 作为一个多平台运营者，我生成文章后选择"发布到公众号"，系统自动将文章格式化并通过公众号API发布为草稿。我在微信后台确认后即可推送。

#### 3.1.3 支持平台（按优先级）

| 平台 | 接入方式 | 发布模式 | 优先级 |
|------|---------|---------|--------|
| **微信公众号** | 公众号API（需用户授权） | 发布为草稿（需用户在后台确认推送） | P2-Phase1 |
| **头条号** | 头条号开放平台API | 发布为草稿 | P2-Phase1 |
| **知乎** | 知乎创作者API | 发布为草稿 | P2-Phase2 |
| **小红书** | 无官方API，提供复制+跳转 | 复制内容→打开小红书网页版 | P2-Phase2 |

#### 3.1.4 功能规格

**平台授权流程**:

```
设置页 → 平台绑定
┌──────────────────────────────────────┐
│  🔗 平台绑定管理                      │
│                                      │
│  微信公众号  [已绑定: xxx的公众号] [解绑]│
│  头条号      [未绑定]        [去绑定]  │
│  知乎        [未绑定]        [去绑定]  │
│  小红书      无需绑定（复制模式）       │
└──────────────────────────────────────┘
```

**发布流程**:

```
文章预览页 → 发布按钮
┌──────────────────────────────────────┐
│  📤 发布文章                          │
│                                      │
│  选择发布平台:                        │
│  ☑ 微信公众号 (已绑定)               │
│  ☑ 头条号 (已绑定)                   │
│  ☐ 知乎 (未绑定，去绑定)             │
│                                      │
│  发布模式: ○ 草稿  ● 定时发布         │
│  定时: [2026-02-16 08:00]            │
│                                      │
│  [📤 发布到选中平台]                  │
│                                      │
│  发布结果:                            │
│  ✅ 微信公众号 — 已保存为草稿          │
│  ✅ 头条号 — 已保存为草稿              │
└──────────────────────────────────────┘
```

#### 3.1.5 后端实现要点

**新增服务**: `backend/api/services/publisher_service.py`

```python
class PublisherService:
    async def publish(self, article_id: int, platforms: list, user_id: int) -> list:
        """
        多平台发布编排:
        1. 获取文章内容
        2. 对每个平台进行格式转换（复用P0 platform_converter）
        3. 调用各平台API发布
        4. 记录发布结果
        """
```

**OAuth管理**: 各平台OAuth token存储和刷新

**新增数据表**: `platform_connections`, `publish_logs`

**新增API**:

```
GET /api/v1/platforms/connections          # 获取已绑定平台
POST /api/v1/platforms/connect/{platform}  # 发起OAuth绑定
DELETE /api/v1/platforms/disconnect/{platform}
POST /api/v1/articles/{article_id}/publish
Request: { platforms: ["wechat", "toutiao"], mode: "draft", scheduled_at: null }
Response: { results: [{ platform, status, message, external_url }] }
GET /api/v1/articles/{article_id}/publish-history
```

#### 3.1.6 验收标准

- [ ] 用户可通过OAuth绑定微信公众号和头条号
- [ ] 绑定后可一键发布文章为草稿
- [ ] 发布时自动应用对应平台格式（复用P0多平台转换）
- [ ] 发布结果实时反馈（成功/失败+原因）
- [ ] 发布历史可查看
- [ ] 支持定时发布（存储定时任务，到时触发）
- [ ] OAuth token自动刷新，过期时提示重新授权

### F14: 私有知识库

#### 3.2.1 功能概述

用户可上传文档（PDF/Word/Markdown/网页URL），系统建立私有知识库。生成文章时可选择基于私有知识库生成，确保内容与企业/个人专业知识一致。

#### 3.2.2 用户故事

> 作为企业内容营销负责人，我上传了公司的产品文档、行业报告和历史文章共50篇。之后生成文章时选择"基于知识库"，AI会引用我们的产品特性和行业数据，生成的内容更专业、更符合品牌调性。

#### 3.2.3 功能规格

**知识库管理**:

| 功能 | 规格 |
|------|------|
| 支持格式 | PDF、Word(.docx)、Markdown、纯文本、网页URL |
| 单文件限制 | ≤ 10MB |
| 总容量 | Pro: 100MB / Ultra: 1GB |
| 文档数量 | Pro: 50篇 / Ultra: 500篇 |
| 向量化 | 上传后自动分块+向量化（复用现有FAISS） |
| 管理 | 上传/删除/搜索/分类 |

**知识库写作**:

```
写作页 → 知识库开关
┌──────────────────────────────────────┐
│  主题: [___________________]          │
│                                      │
│  📚 知识库: [✅ 启用]                 │
│  选择知识库: [全部] [产品文档] [行业报告]│
│                                      │
│  [🚀 生成文章]                        │
└──────────────────────────────────────┘
```

生成时，系统先从知识库中检索相关内容（语义搜索），将检索结果作为上下文注入到文章生成prompt中（RAG模式）。

#### 3.2.4 后端实现要点

**复用现有能力**: 
- FAISS语义搜索（`faiss_cache.py`已有）
- 文档解析可用 `unstructured` 或 `pymupdf` 库

**新增服务**: `backend/api/services/knowledge_base.py`

**新增数据表**: `knowledge_bases`, `knowledge_documents`, `document_chunks`

**新增API**:

```
POST /api/v1/knowledge-base/upload     # 上传文档
GET /api/v1/knowledge-base/documents   # 文档列表
DELETE /api/v1/knowledge-base/documents/{id}
POST /api/v1/knowledge-base/search     # 语义搜索（调试用）
```

**article_worker修改**: 生成时检查用户是否启用知识库，如启用则先检索再生成

#### 3.2.5 验收标准

- [ ] 支持PDF/Word/Markdown/纯文本/URL五种格式上传
- [ ] 上传后自动向量化（进度可见）
- [ ] 写作页可开启/关闭知识库
- [ ] 启用知识库后，生成的文章引用知识库中的信息
- [ ] 知识库搜索相关性准确（Top-5命中率>70%）
- [ ] 容量限制正确（Pro 100MB / Ultra 1GB）
- [ ] 文档可删除，删除后向量索引同步更新

### F15: 团队协作

#### 3.3.1 功能概述

支持多人团队使用，包括团队创建、成员管理、文章共享、品牌风格统一、审批流程。

#### 3.3.2 用户故事

> 作为内容团队负责人，我创建了一个5人团队。团队成员共享知识库和品牌风格，每人生成的文章自动进入审批队列。我审核通过后才能发布。团队共享一个Ultra订阅，按席位付费。

#### 3.3.3 功能规格

**团队管理**:

| 功能 | 规格 |
|------|------|
| 创建团队 | 团队名称、Logo、描述 |
| 成员角色 | Owner（1人）、Admin、Editor、Viewer |
| 成员上限 | 按席位付费，¥199/人/月 |
| 邀请方式 | 邮件邀请链接 |

**权限矩阵**:

| 操作 | Owner | Admin | Editor | Viewer |
|------|-------|-------|--------|--------|
| 团队设置 | ✅ | ✅ | ❌ | ❌ |
| 成员管理 | ✅ | ✅ | ❌ | ❌ |
| 品牌风格设置 | ✅ | ✅ | ❌ | ❌ |
| 生成文章 | ✅ | ✅ | ✅ | ❌ |
| 审批文章 | ✅ | ✅ | ❌ | ❌ |
| 查看文章 | ✅ | ✅ | ✅ | ✅ |
| 发布文章 | ✅ | ✅ | ❌ | ❌ |

**审批流程**:

```
Editor生成文章 → 提交审批 → Admin/Owner审核 → 通过→可发布 / 驳回→修改
```

**品牌风格统一**: 团队级写作风格（复用P0风格学习），所有成员生成时自动应用

#### 3.3.4 后端实现要点

**新增数据表**: `teams`, `team_members`, `team_invitations`, `article_approvals`

**新增API**:

```
POST /api/v1/teams/create
GET /api/v1/teams/my-team
PUT /api/v1/teams/{team_id}
POST /api/v1/teams/{team_id}/invite
POST /api/v1/teams/{team_id}/members/{user_id}/role
DELETE /api/v1/teams/{team_id}/members/{user_id}
POST /api/v1/articles/{article_id}/submit-review
POST /api/v1/articles/{article_id}/approve
POST /api/v1/articles/{article_id}/reject
GET /api/v1/teams/{team_id}/pending-reviews
```

#### 3.3.5 验收标准

- [ ] 可创建团队并邀请成员
- [ ] 4种角色权限控制正确
- [ ] 团队成员共享知识库
- [ ] 团队品牌风格自动应用
- [ ] 审批流程完整（提交→审核→通过/驳回）
- [ ] 团队按席位计费

### F16: 内容日历

#### 3.4.1 功能概述

提供内容排期管理工具，支持日历视图查看已发布/待发布/草稿文章，节日热点提前提醒，帮助运营者规划内容节奏。

#### 3.4.2 用户故事

> 作为一个公众号运营者，我打开内容日历看到本周已安排3篇文章，周五还空着。日历提醒我下周一是情人节，建议提前准备相关内容。我点击周五的空位，直接跳转到写作页开始创作。

#### 3.4.3 功能规格

**日历视图**:

| 功能 | 规格 |
|------|------|
| 视图模式 | 月视图 / 周视图 |
| 文章状态 | 草稿(灰) / 待发布(蓝) / 已发布(绿) / 定时发布(橙) |
| 拖拽排期 | 拖拽文章到指定日期 |
| 快速创建 | 点击空白日期→跳转写作页 |
| 节日提醒 | 中国节日/节气/营销节点自动标注 |
| 热点预测 | 基于历史数据预测下周可能的热点领域 |

**节日/营销节点数据库**: 内置中国主要节日、24节气、电商节日（618/双11等）、行业纪念日

#### 3.4.4 后端实现要点

**新增服务**: `backend/api/services/calendar_service.py`

**新增数据表**: `content_calendar` (排期记录), `calendar_events` (节日/事件)

**新增API**:

```
GET /api/v1/calendar?month=2026-02
Response: {
    articles: [{ date, article_id, title, status }],
    events: [{ date, name, type, suggestions }]
}
POST /api/v1/calendar/schedule
Request: { article_id, scheduled_date }
PUT /api/v1/calendar/reschedule
```

#### 3.4.5 验收标准

- [ ] 月视图/周视图切换正常
- [ ] 文章按状态颜色标注
- [ ] 可拖拽文章到指定日期
- [ ] 点击空白日期跳转写作页
- [ ] 节日/营销节点自动标注（至少覆盖50个主要节点）
- [ ] Pro+用户可用

### F17: API开放

#### 3.5.1 功能概述

开放文章生成API，供第三方开发者集成SupaWriter的写作能力到自己的产品中。

#### 3.5.2 功能规格

**API能力**:

| API | 说明 | 计费 |
|-----|------|------|
| `POST /open/v1/articles/generate` | 生成文章（同步/异步） | 按次计费 |
| `GET /open/v1/articles/{task_id}/status` | 查询生成状态 | 免费 |
| `POST /open/v1/articles/score` | 文章评分 | 按次计费 |
| `POST /open/v1/articles/convert` | 平台格式转换 | 按次计费 |
| `POST /open/v1/seo/analyze` | SEO分析 | 按次计费 |

**认证**: API Key（已有 `api_keys` 表和加密管理）

**计费**: 按调用量计费，Ultra用户每月赠送1000次API调用

**文档**: 自动生成OpenAPI/Swagger文档

#### 3.5.3 后端实现要点

新增 `/open/v1/` 路由前缀，复用现有服务层，增加API Key认证中间件和调用量计费。

**新增数据表**: `api_usage_logs`

#### 3.5.4 验收标准

- [ ] Ultra用户可在设置页创建API Key
- [ ] API Key认证正常
- [ ] 5个开放API端点可用
- [ ] 调用量统计准确
- [ ] Swagger文档自动生成
- [ ] 超出配额返回429错误
