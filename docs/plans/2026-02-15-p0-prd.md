## 一、产品背景

SupaWriter 已具备"热点追踪→智能选题→搜索增强写作→微信预览"的核心链路，但缺乏以下爆款产品要素：
1. **Aha Moment 不够强烈** — 用户需手动从热点页复制标题再到写作页粘贴，链路断裂
2. **无自传播机制** — 生成的文章没有任何品牌露出，无法形成增长飞轮
3. **平台适配单一** — 仅支持微信预览，知乎/小红书/头条用户无法直接使用
4. **缺乏质量反馈** — 用户不知道生成的文章质量如何，没有改进方向
5. **无个性化能力** — 所有用户生成的文章风格相同，缺乏差异化

### F1: 爆文生成器（热点→一键写作）

#### 2.1.1 功能概述

在热点追踪页面，为每条热点新闻增加"写这个"按钮，用户点击后自动跳转到写作页面并预填主题，实现"看到热点→一键开始写作"的极致体验。

#### 2.1.2 用户故事

> 作为一个公众号运营者，我在热点页看到一条热搜，点击"写这个"按钮，系统自动跳转到写作页并填好主题，我只需点击"生成"就能得到一篇蹭热点的文章。

#### 2.1.3 功能规格

**前端 — 热点追踪页 (`frontend/src/app/hotspots/` 或对应页面)**

| 元素 | 规格 |
|------|------|
| 按钮位置 | 每条热点卡片右侧，与标题同行 |
| 按钮文案 | "✍️ 写这个" |
| 按钮样式 | Ghost/outline 样式，hover时高亮，不抢占热点标题视觉焦点 |
| 点击行为 | 1. 将热点标题存入URL参数或状态管理 2. 路由跳转到 `/writer?topic={热点标题}&source=hotspot` |

**前端 — 写作页 (`frontend/src/app/writer/page.tsx`)**

| 元素 | 规格 |
|------|------|
| 参数接收 | 从URL query参数读取 `topic` 和 `source` |
| 主题预填 | 自动填入主题输入框，用户可修改 |
| 来源标识 | 如果 `source=hotspot`，在主题输入框下方显示小标签："📌 来自热点追踪" |
| 自动聚焦 | 预填后自动聚焦到"生成"按钮，引导用户一键生成 |

**后端 — 无需新增API**

热点数据已有，写作API已有，本功能仅需前端串联。

#### 2.1.4 交互流程

```
热点追踪页
  └─ 用户看到热点列表
      └─ 点击某条热点的"✍️ 写这个"
          └─ 路由跳转: /writer?topic=热点标题&source=hotspot
              └─ 写作页自动预填主题
                  └─ 用户点击"生成文章"
                      └─ 正常文章生成流程
```

#### 2.1.5 验收标准

- [ ] 热点页每条热点右侧显示"✍️ 写这个"按钮
- [ ] 点击后跳转到写作页，主题输入框自动填入热点标题
- [ ] 显示"来自热点追踪"来源标签
- [ ] 用户可修改预填的主题
- [ ] 从点击按钮到写作页加载完成 < 1秒
- [ ] 移动端适配：按钮不溢出，可正常点击

### F2: 文章品牌水印

#### 2.2.1 功能概述

免费版用户生成的文章，在导出/预览时自动在底部添加 SupaWriter 品牌水印和注册链接，实现"内容即广告"的增长飞轮。付费用户（Pro/Ultra）可去除水印。

#### 2.2.2 用户故事

> 作为免费用户，我生成文章后复制到公众号发布，文章底部会有一行小字"本文由 SupaWriter AI 辅助创作 | supawriter.com"。
> 作为Pro用户，我生成的文章没有水印。

#### 2.2.3 功能规格

**水印内容**

```
---
本文由 [SupaWriter](https://supawriter.com) AI 辅助创作
```

**水印规则**

| 场景 | Free用户 | Pro/Ultra用户 |
|------|---------|-------------|
| Markdown预览 | 显示水印 | 不显示 |
| 微信/多平台预览 | 显示水印（HTML格式） | 不显示 |
| 下载Markdown文件 | 包含水印 | 不包含 |
| 编辑器内编辑 | 不显示水印（编辑时不干扰） | 不显示 |

**水印样式（HTML版本）**

```html
<div style="margin-top: 40px; padding-top: 20px; border-top: 1px solid #e0e0e0; text-align: center; color: #999; font-size: 12px;">
  本文由 <a href="https://supawriter.com?ref=watermark" style="color: #10b981;">SupaWriter</a> AI 辅助创作
</div>
```

**后端实现要点**

- 水印在文章**输出时注入**，不存入数据库原文
- 通过 `TierService.get_user_tier()` 判断用户等级
- 在文章预览/下载的API响应中条件性添加水印

#### 2.2.4 验收标准

- [ ] 免费用户：Markdown预览底部显示水印
- [ ] 免费用户：微信预览底部显示水印（HTML格式，带链接）
- [ ] 免费用户：下载的Markdown文件包含水印
- [ ] Pro/Ultra用户：所有场景均无水印
- [ ] 水印链接带 `?ref=watermark` 追踪参数
- [ ] 水印视觉不突兀，与文章内容有明显分隔

### F3: 多平台格式适配

#### 2.3.1 功能概述

扩展现有微信预览能力，新增知乎、小红书、头条三个平台的格式适配，用户可一键将文章转换为目标平台的最佳格式并复制。

#### 2.3.2 用户故事

> 作为一个多平台运营者，我生成一篇文章后，可以在预览页选择"知乎"平台，系统自动调整格式（加粗重点、调整段落间距、添加话题标签），我一键复制后粘贴到知乎即可发布。

#### 2.3.3 各平台格式规格

**微信公众号（已有，保持不变）**

| 特征 | 规格 |
|------|------|
| 格式 | 内联CSS的HTML |
| 标题 | 居中，大字号 |
| 段落 | 首行缩进2字符，行高1.8 |
| 图片 | 居中，圆角，max-width:100% |
| 引用 | 左侧绿色边框 |
| 代码 | 灰色背景块 |

**知乎**

| 特征 | 规格 |
|------|------|
| 格式 | 纯Markdown（知乎编辑器原生支持） |
| 标题 | 使用 ## 二级标题（知乎不支持一级标题） |
| 加粗 | 关键观点和结论加粗 |
| 分隔 | 章节间用 `---` 分隔 |
| 话题标签 | 文末自动生成3-5个知乎话题标签建议 |
| 引用 | 数据来源用 `>` 引用格式 |

**小红书**

| 特征 | 规格 |
|------|------|
| 格式 | 纯文本 + emoji |
| 标题 | emoji开头，如 "🔥 标题内容" |
| 段落 | 短段落（每段≤3句），段间空行 |
| 要点 | 用 emoji bullet 替代列表符号（✅ ❌ 💡 📌） |
| 话题标签 | 文末生成 #话题# 格式标签（5-10个） |
| 字数 | 控制在800-1200字 |
| 风格 | 口语化、亲切感、适当使用感叹号 |

**头条号**

| 特征 | 规格 |
|------|------|
| 格式 | HTML（头条编辑器支持富文本粘贴） |
| 标题 | 吸引眼球的标题（可自动优化） |
| 段落 | 短段落，每段2-3句 |
| 加粗 | 关键数字和结论加粗 |
| 图片 | 保留，居中显示 |
| 小标题 | 使用加粗而非标题标签 |

### F4: 内容质量评分

#### 2.4.1 功能概述

文章生成完成后，自动对文章进行多维度质量评分，帮助用户了解文章质量并提供改进建议。免费版显示总分，Pro版显示详细维度分数和优化建议。

#### 2.4.2 用户故事

> 作为免费用户，文章生成后我看到一个质量评分"78/100"和一个简短评语"文章结构清晰，建议补充更多数据支撑"。
> 作为Pro用户，我还能看到可读性、SEO、传播力、原创度四个维度的详细分数和每个维度的具体优化建议。

#### 2.4.3 评分维度

| 维度 | 权重 | 评分标准 | 分值范围 |
|------|------|---------|---------|
| **可读性** | 30% | 段落长度、句子复杂度、过渡词使用、标题层次 | 0-100 |
| **信息密度** | 25% | 数据引用、事实支撑、论据充分性 | 0-100 |
| **SEO友好度** | 25% | 标题吸引力、关键词分布、元描述质量、内链潜力 | 0-100 |
| **传播潜力** | 20% | 标题吸引力、开头hook、情感共鸣、分享动机 | 0-100 |

**总分计算**: `总分 = 可读性×0.3 + 信息密度×0.25 + SEO×0.25 + 传播力×0.2`

#### 2.4.4 评分等级

| 分数段 | 等级 | 颜色 | 评语模板 |
|--------|------|------|---------|
| 90-100 | 🏆 优秀 | 绿色 | "高质量文章，可直接发布" |
| 75-89 | 👍 良好 | 蓝色 | "质量不错，稍作优化即可发布" |
| 60-74 | 📝 一般 | 黄色 | "建议根据优化建议修改后发布" |
| 0-59 | ⚠️ 待改进 | 红色 | "建议重新生成或大幅修改" |

### F5: 写作风格学习

#### 2.5.1 功能概述

用户可上传3-5篇自己的代表作品，系统分析其写作风格特征（用词偏好、句式结构、段落节奏、语气风格等），在后续文章生成时应用该风格，实现个性化写作。

#### 2.5.2 用户故事

> 作为一个有固定写作风格的博主，我上传了5篇我的代表文章，系统分析出我的风格是"理性分析+数据驱动+适度幽默"。之后生成的文章都会带有我的风格特征，读者看不出是AI写的。

#### 2.5.3 功能规格

**风格分析流程**:

```
用户上传3-5篇范文（Markdown/纯文本）
    ↓
系统分析风格特征（LLM）
    ↓
生成风格描述文件（JSON）
    ↓
存储到用户风格库
    ↓
后续生成文章时，将风格描述注入system prompt
```

**风格特征维度**:

| 维度 | 分析内容 | 示例 |
|------|---------|------|
| **语气风格** | 正式/口语/幽默/严肃/亲切 | "偏口语化，适度使用网络用语" |
| **句式偏好** | 长句/短句、主动/被动、设问/反问 | "偏好短句，常用设问句引导思考" |
| **段落结构** | 段落长度、过渡方式、论证模式 | "每段3-5句，先抛观点再举例" |
| **用词特征** | 专业术语使用、修辞偏好 | "善用比喻，少用成语" |
| **开头风格** | 开篇方式 | "常以故事或场景描写开头" |
| **结尾风格** | 收尾方式 | "常以总结+行动号召结尾" |

**前端交互**:

```
设置页 → 写作风格
┌─────────────────────────────────────┐
│  ✍️ 我的写作风格                     │
│                                     │
│  上传您的代表作品，AI将学习您的风格    │
│                                     │
│  📄 我的公众号爆款文章.md    ✕       │
│  📄 知乎高赞回答.md          ✕       │
│  📄 行业分析报告.md          ✕       │
│                                     │
│  [+ 添加范文]  (3/5篇，至少3篇)      │
│                                     │
│  [🔍 分析我的风格]                    │
│                                     │
│  ── 风格分析结果 ──                  │
│  语气: 理性分析型，适度幽默           │
│  句式: 偏好短句，善用设问             │
│  结构: 总分总，数据驱动               │
│  特征: 善用比喻解释复杂概念           │
│                                     │
│  ✅ 已启用 — 后续生成将应用此风格      │
│  [重新分析] [关闭风格]                │
└─────────────────────────────────────┘
```

**范文要求**:
- 格式：Markdown 或纯文本
- 字数：每篇 ≥ 500字
- 数量：3-5篇
- 大小：单篇 ≤ 50KB

### F6: 新用户引导优化

#### 2.6.1 功能概述

优化新用户注册后的首次使用体验，通过引导流程帮助用户在3分钟内完成首篇文章生成，提升"注册→首篇文章"的转化率。

#### 2.6.2 用户故事

> 作为一个刚注册的新用户，我登录后看到一个简洁的引导页面，告诉我"3步生成你的第一篇AI文章"，我跟着引导选了一个热点话题，点击生成，2分钟后就看到了一篇完整的文章。我觉得这个工具很有用。

#### 2.6.3 引导流程设计

**触发条件**: 用户首次登录（`articles` 表中该用户文章数 = 0）

**引导步骤**:

```
Step 1: 欢迎 + 选择身份
┌─────────────────────────────────────┐
│  👋 欢迎来到 SupaWriter!             │
│                                     │
│  告诉我们你是谁，我们为你推荐最佳体验  │
│                                     │
│  [📱 自媒体运营者]                    │
│  [💼 企业内容营销]                    │
│  [✍️ 自由撰稿人]                     │
│  [🌟 个人IP/副业]                    │
│                                     │
│  [跳过，直接开始 →]                   │
└─────────────────────────────────────┘

Step 2: 选择一个热点话题
┌─────────────────────────────────────┐
│  🔥 选一个感兴趣的热点，体验AI写作    │
│                                     │
│  (展示当前3-5条热门热点，可点击选择)   │
│                                     │
│  ○ 热点话题A                         │
│  ○ 热点话题B                         │
│  ● 热点话题C  ← 已选                 │
│                                     │
│  或者输入自定义主题: [____________]    │
│                                     │
│  [← 上一步]  [开始生成 →]            │
└─────────────────────────────────────┘

Step 3: 生成中 + 结果展示
┌─────────────────────────────────────┐
│  ⏳ 正在为你生成文章...               │
│                                     │
│  (复用现有TaskProgress组件)           │
│                                     │
│  生成完成后:                          │
│  🎉 你的第一篇AI文章已生成!           │
│                                     │
│  [查看文章] [再写一篇] [探索更多功能]  │
└─────────────────────────────────────┘
```

#### 2.6.4 实现要点

- 引导状态存储在 `localStorage`（`onboarding_completed: true`）
- 引导页为独立路由 `/onboarding` 或 modal overlay
- 用户身份选择存储到 `user_preferences` 表（可选，用于后续个性化推荐）
- 热点数据复用现有 hotspots API
- 文章生成复用现有 writer 流程

#### 2.6.5 验收标准

- [ ] 新用户首次登录自动进入引导流程
- [ ] 引导流程可跳过
- [ ] 完成引导后不再重复出现
- [ ] 从引导开始到首篇文章生成 < 3分钟（不含文章生成等待时间）
- [ ] 引导页移动端适配良好
- [ ] 引导中的热点数据为实时数据

## 三、非功能需求

### 3.1 性能要求

| 场景 | 指标 | 目标值 |
|------|------|--------|
| 页面加载 | 首屏渲染时间 | < 2秒 |
| API响应 | 非LLM接口 | < 500ms |
| 平台转换 | 微信/知乎/头条 | < 500ms |
| 平台转换 | 小红书（含LLM改写） | < 10秒 |
| 质量评分 | 评分完成时间 | < 10秒 |
| 风格分析 | 分析完成时间 | < 30秒 |

### 3.2 兼容性要求

- 浏览器：Chrome 90+, Safari 15+, Firefox 90+, Edge 90+
- 移动端：≥ 375px 宽度正常显示和交互
- 系统：Windows/macOS/iOS/Android

### 3.3 数据安全

- 用户上传的范文仅用于风格分析，不对外暴露
- 文章内容存储在用户自己的数据空间，其他用户不可访问
- 品牌水印不包含用户个人信息
