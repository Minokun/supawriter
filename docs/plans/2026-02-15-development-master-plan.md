## 六、技术约束与前提

### 6.1 已有能力（可直接复用）

| 能力 | 现有实现 | 复用阶段 |
|------|---------|---------|
| 热点数据获取 | `hotspots_service.py` — 5平台爬虫+Redis缓存 | P0/P1 |
| 文章生成 | `article_worker.py` — arq异步任务 | P0/P1/P2 |
| 微信格式转换 | `wechat_converter.py` — Markdown→微信HTML | P0 |
| FAISS语义搜索 | `faiss_cache.py` — 向量索引+检索 | P2 知识库 |
| Embedding | `embedding_utils.py` — 多模型向量化 | P2 知识库 |
| WebSocket | `manager.broadcast()` — 实时推送 | P1 通知/P2 团队 |
| LLM调用 | `utils/llm_chat.py` — 多模型支持 | 全阶段 |
| 会员等级 | `tier_service.py` — Free/Pro/Ultra | P1 付费体系 |
| API Key管理 | `api_keys` 表 + 加密存储 | P2 API开放 |

### 6.2 技术栈

- **前端**: Next.js 14 + React 18 + TailwindCSS + shadcn/ui
- **后端**: FastAPI + SQLAlchemy + arq worker
- **数据库**: PostgreSQL + Redis
- **AI**: 多LLM支持（DeepSeek/GPT/Claude）+ FAISS向量搜索
- **部署**: Docker Compose

### 6.3 数据库迁移文件

| 文件 | 内容 | 阶段 |
|------|------|------|
| `008_p0_features.sql` | 写作风格、文章评分、用户引导（3表） | P0 |
| `009_p1_features.sql` | 批量任务、Agent、预警、通知、订单、订阅、额度包（9表） | P1 |
| `010_p2_features.sql` | 平台绑定、发布日志、知识库、团队、日历、API日志（12表） | P2 |
| `011-013` (规划) | 企业版、社区、插件生态 | P3 |

## 九、风险总览

| 风险 | 影响阶段 | 缓解措施 |
|------|---------|---------|
| LLM Prompt效果不稳定 | P0/P1 | 预留调优时间，准备基准测试集 |
| 多平台格式差异大 | P0 | 优先做公众号+知乎，其他延后 |
| 微信支付审核延迟 | P1 | 先用支付宝或手动充值过渡 |
| 公众号API审核延迟 | P2 | 先做头条号，公众号延后 |
| 团队功能复杂度超预期 | P2 | 先上基础版（无审批），审批后续迭代 |
| 企业客户获取困难 | P3 | 先做种子客户，打磨后再推广 |
