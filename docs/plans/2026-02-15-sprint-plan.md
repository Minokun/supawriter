## 二、Sprint 1 — 爆文生成器 + 水印 (Week 1)

### 2.1 目标

让用户可以从热点页一键跳转到写作页生成文章，免费版文章自动带品牌水印。

### 2.2 任务拆分

| # | 任务 | 负责 | 工时 | 依赖 | 产出 |
|---|------|------|------|------|------|
| 1.1 | 数据库迁移：执行 `008_p0_features.sql` | 后端 | 0.5天 | 无 | 3张新表创建完成 |
| 1.2 | 后端：水印注入工具 `watermark.py` | 后端 | 0.5天 | 无 | 水印工具+单元测试 |
| 1.3 | 后端：文章预览/下载API集成水印 | 后端 | 0.5天 | 1.2 | 预览和下载接口条件性添加水印 |
| 1.4 | 前端：热点页添加"写这个"按钮 | 前端 | 1天 | 无 | 每条热点右侧显示按钮 |
| 1.5 | 前端：写作页接收URL参数预填主题 | 前端 | 0.5天 | 1.4 | topic参数预填+来源标签 |
| 1.6 | 前端：水印在预览/下载中展示 | 前端 | 0.5天 | 1.3 | Free用户看到水印，Pro不看到 |
| 1.7 | 联调测试：爆文生成器完整流程 | 全员 | 0.5天 | 1.4, 1.5 | 热点→写作→生成 E2E通过 |

### 2.3 每日计划

| 日期 | 后端 | 前端 | 验收点 |
|------|------|------|--------|
| **Day 1** | 1.1 数据库迁移 + 1.2 水印工具 | 1.4 热点页"写这个"按钮 | 数据库表创建成功 |
| **Day 2** | 1.3 API集成水印 | 1.5 写作页参数预填 | 热点→写作跳转可用 |
| **Day 3** | 代码review + bug修复 | 1.6 水印展示 + 1.7 联调 | **M1达成**: 爆文+水印可演示 |

### 2.4 M1 验收 Checklist

- [ ] 热点页每条热点显示"✍️ 写这个"按钮
- [ ] 点击后跳转到 `/writer?topic=xxx&source=hotspot`

- [ ] 显示"📌 来自热点追踪"来源标签

- [ ] Free用户：微信预览底部有水印（HTML格式）

- [ ] Pro/Ultra用户：所有场景无水印
- [ ] 水印链接带 `?ref=watermark` 参数
- [ ] 移动端适配正常

## 七、发布后监控

| 指标 | 监控方式 | 预警阈值 | 负责人 |
