Listener 与 Writer 跑在 **Tokio 多线程调度器**，完全绕过 GIL - Handler 只在真正调用 Python 函数时拿锁，**持有时间 < 1 µs** - 通道采用 **crossbeam::mpsc**，无锁 CAS 实现，单队列 8 线程可飙到 1200 万 msg/s > 压测 8 核机器，FastAPI 只能把 CPU 吃到 120%，Robyn 直接干到 780%，**GIL 形同虚设**。 --- ### WebSocket/HTTP1/2统一事件循环：长连接不再开线程 常见“异步”框架的真相： - Flask-SocketIO：每个 WebSocket 背后默认 1 个线程 → 1 万连接 1 万个线程，内存爆炸 - FastAPI + Uvicorn：HTTP 与 WebSocket 事件循环分离，**双循环上下文切换**带来 15% 性能税 Robyn 把 **HTTP/1.1、HTTP/2、WebSocket** 全部塞进 **单 Tokio Runtime**： - 统一基于 `tokio::net::TcpStream` 与 `hyper::upgrade` -
